About — FSM Theory ▼

This is a fully functional Navi Mumbai Metro ATVM simulator modelled as a Finite State Machine (FSM). The machine processes user inputs and transitions between states to complete a full ticket booking lifecycle.

The FSM has 10 visual states: Q0 (Idle/Menu) → Q1 (User Type) → Q2 (Card Validation) → Q3 (Destination) → Q4 (Journey Type) → Q5 (Passengers/Fare) → Q6 (Payment) → Q7 (Print ✓). Error state QI handles invalid inputs; QR handles cancel, recharge, and info branches. The live diagram above is always in sync with the machine's internal state.
